# 01. プロジェクト概要

## 1. プロジェクト名

**FrontierAtlas**

## 2. プロジェクトの目的

FrontierAtlas は、**スマートフォンのみで動作する屋内測位システム**の研究・実装プロジェクトである。

GPS に依存せず、以下の技術を利用して建物内部における利用者の位置を推定する：

- Wi-Fi Fingerprint
- 地磁気 Fingerprint
- PDR（Pedestrian Dead Reckoning）
- スマートフォン内蔵センサー（加速度・ジャイロ・コンパス）

研究で得られた成果を、そのまま実際の施設で利用できる汎用的な屋内マップシステムへ発展させることを目標とする。

## 3. 研究背景

GPS は屋外では高精度だが、建物内部・地下施設・商業施設・学校・病院などでは十分な精度を得られない。既存の屋内測位システム（BLE Beacon / UWB / 独自センサー）は専用設備が必要で導入コストが高い。

本研究では、**既存建物に専用設備を追加せず、スマートフォンだけで利用可能な屋内測位システム**の実現を目指す。

## 4. 研究目的

1. **屋内測位アルゴリズムの研究**: Android 端末の加速度・ジャイロ・地磁気・Wi-Fi を統合し、高精度な位置推定を実現する。
2. **汎用マップシステムの構築**: 研究専用ではなく、実際の施設で利用可能な地図表示・ナビゲーション・POI 表示・経路表示を備えた汎用屋内マップシステムを構築する。
3. **データ更新基盤の構築**: アプリ更新を必要とせず、地図・POI・ナビゲーション情報・測位関連データを更新可能なシステムを構築する。

## 5. 基本設計思想

1. **責務分離**: UI・地図描画・測位・データ更新を独立して実装する。
2. **レイヤ構造**: 上位レイヤは下位レイヤの詳細実装を意識しない。Android 測位エンジンの内部変更が React Native 側に波及しない。
3. **データ駆動設計**: 建物情報をコードに埋め込まず、すべて GeoJSON・metadata・navigation・Wi-Fi・地磁気データから構築する。建物追加はデータ追加のみで完結。
4. **オフラインファースト**: ネットワーク接続なしでも地図表示・ナビゲーション・測位が利用可能。通信は更新時のみ。
5. **クロスプラットフォーム**: 研究は Android 上で行うが、UI は React Native を採用し、Android・iOS・Web へ展開可能な構造とする。

## 6. 座標系

本プロジェクトでは **GPS 座標（WGS84）とローカル座標系の2種類を併用**する。Map Assets Pipeline が両方を生成し同一リリースに同梱する。各コンポーネントは自身の責務に適した座標系のデータを Map Assets から直接読み込む。

> **詳細は `03_system-architecture.md` §6「座標管理」を参照**。座標系の定義・コンポーネント別の使用座標・実行時変換ルール・TODO事項は 03 に集約する。

## 7. システムの基本構成

システムは以下の3つの主要コンポーネントから構成される：

| # | コンポーネント | 1行説明 |
|---|--------------|---------|
| ① | React Native Application | UI・地図表示・ユーザー操作・ナビゲーション表示 |
| ② | Android Position Engine | センサー取得・歩行推定・位置推定・フィルタリング |
| ③ | Map Assets Pipeline | GeoJSON・Wi-Fi・地磁気・POI・metadata の建物情報を保持 / 変換・リリースを担当 |

## 8. 基本データフロー

1. MapLibre が地図を描画
2. Kotlin 測位エンジンが位置を推定
3. React Native が位置情報を受け取り UI を更新
